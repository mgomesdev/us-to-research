## UC02 - Selecionar Responsável pelos Ingressos

- **Caso de Uso Geral:** Checkout
- **Ator Principal:** Participante
- **Atores Secundários:** -
- **Resumo:** Este caso de uso representa o processo de seleção do responsável pelos ingressos no Bloco 2 do checkout. O participante pode escolher entre os participantes informados no Bloco 1 ou selecionar "Outro" para informar dados manualmente. O bloco também inclui aceite de termos e condições.
- **Pré-condições:** Bloco 1 preenchido (para checkout nominal)
- **Pós-condições:** Responsável pelosingressos definido e termos aceitos

### Cenário Principal

| Ações do Ator | Ações do Sistema |
|---------------|------------------|
| 1. Acessa o Bloco 2 - Recebimento do ingresso | 2. Exibe dropdown "Responsável pelos ingressos" com nomes dos participantes + "Outro" |
| 3. Seleciona um participante da lista | 4. Usa automaticamente o e-mail daquele participante como destinatário |
| | 5. Não exibe campos adicionais |
| | 6. Exibe checkboxes de termos |
| 7. Marca "Eu li e concordo com os termos e condições" | 8. Exibe nota informativa sobre criação automática de conta |
| 9. Opcional: marca "Aceito receber contato e informações sobre o evento" | |
| 10. Clica em "Avançar" | 11. Valida aceite obrigatório e permite avanço |

### Cenário Alternativo

**Selecionar "Outro":**

| Ações do Ator | Ações do Sistema |
|---------------|------------------|
| 1. Seleciona "Outro" no dropdown | 2. Exibe campos: nome completo, e-mail, celular (obrigatórios) |
| 3. Preenche os campos | 4. Valida dados |

**Checkout não nominal:**

| Ações do Ator | Ações do Sistema |
|---------------|------------------|
| | 1. Bloco 2 não exibe dropdown |
| | 2. Exibe apenas checkboxes e nota informativa |

### Cenário de Exceção

- Checkbox de termos não marcado: exibir mensagem de erro e bloquear avanço

### Requisitos Não-Funcionais

- Dropdown deve listar todos os participantes do Bloco 1
