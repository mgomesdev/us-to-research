## UC08 - Finalizar Checkout com Boleto

- **Caso de Uso Geral:** Checkout
- **Ator Principal:** Participante
- **Atores Secundários:** Sistema Financeiro
- **Resumo:** Este caso de uso representa o processo de pagamento via Boleto. O sistema gera o boleto e redireciona para tela de "Aguardando pagamento".
- **Pré-condições:** Dados do comprador preenchidos e Boleto selecionado
- **Pós-condições:** Boleto gerado e aguardando compensação

### Cenário Principal

| Ações do Ator | Ações do Sistema |
|---------------|------------------|
| 1. Seleciona "Boleto" | 2. Gera boleto |
| | 3. Exibe tela de "Aguardando pagamento" |
| | 4. Permite visualizar instruções e baixar boleto |
| | 5. Permite copiar código do boleto |
| | 6. Aguarda compensação |

### Cenário Alternativo

**Após compensação:**

| Ações do Ator | Ações do Sistema |
|---------------|------------------|
| | 1. Recebe webhook de confirmação |
| | 2. Exibe tela de confirmação |

### Cenário de Exceção

- Boleto não pago após vencimento: liberar ingressos

### Requisitos Não-Funcionais

- Tempo de compensação deve ser considerado
- Exibir informações claras de如何 pagamentopara o usuário
