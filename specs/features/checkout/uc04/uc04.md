## UC04 - Preencher Dados do Comprador

- **Caso de Uso Geral:** Checkout
- **Ator Principal:** Participante
- **Atores Secundários:** -
- **Resumo:** Este caso de uso representa o preenchimento dos dados do comprador no Bloco 3. O participante pode selecionar um dos participantes informados no Bloco 1 ou escolher "Outro" para preenchimento manual. Para usuários logados com dados salvos, o sistema pré-preenche automaticamente.
- **Pré-condições:** Método de pagamento selecionado
- **Pós-condições:** Dados do comprador preenchidos e validados

### Cenário Principal

| Ações do Ator | Ações do Sistema |
|---------------|------------------|
| 1. Acessa seção de dados do comprador | 2. Exibe dropdown "Responsável pelo pagamento" com participantes + "Outro" |
| 3. Seleciona um participante | 4. Pré-preenche campos com dados do Bloco 1 |
| 5. Edita dados se necessário | |
| | 7. Exibe opção "Salvar dados para futuras compras" (se logado) |
| 8. Clica em "Confirmar compra" | 9. Valida dados e processa pagamento |

### Cenário Alternativo

**Selecionar "Outro":**

| Ações do Ator | Ações do Sistema |
|---------------|------------------|
| 1. Seleciona "Outro" | 2. Exibe campos para preenchimento manual |

**Usuário logado com dados salvos:**

| Ações do Ator | Ações do Sistema |
|---------------|------------------|
| | 1. Pré-preenche dados salvos automaticamente |
| 2. Pode editar campos | 3. Atualiza dados salvos se houver alteração |
| 4. Desmarca "Salvar dados" | 5. Remove dados salvos após compra |

**Checkout não nominal:** Dropdown não é exibido

### Restrições/Validações

- Dados a salvar: nome completo, e-mail, CPF/CNPJ/Passaporte, telefone
- NÃO salvar: dados do cartão (apenas card_id)
- Dados vinculados ao usuário, não ao método de pagamento
