## UC10 - Criar Conta Automaticamente

- **Caso de Uso Geral:** Checkout
- **Ator Principal:** Participante
- **Atores Secundários:** -
- **Resumo:** Este caso de uso representa a criação automática de conta na plataforma após conclusão bem-sucedida do checkout. A conta é criada com os dados do responsável pelos ingressossem necessidade de autenticação prévia.
- **Pré-condições:** Checkout concluído com sucesso
- **Pós-condições:** Conta criada e e-mails enviados

### Cenário Principal

| Ações do Ator | Ações do Sistema |
|---------------|------------------|
| 1. Conclui checkout com sucesso | 2. Identifica responsável pelos ingressos (Bloco 2) |
| | 3. Verifica se e-mail já possui conta |
| | 4. E-mail não existe: cria conta automaticamente |
| | 5. E-mail existe: usa conta existente |
| | 6. Vincula ingressosao responsável |
| | 7. Envia e-mail de boas-vindas com link de criação de senha |
| | 8. Envia e-mail com ingressos após confirmação do pagamento |

### Cenário Alternativo

**Checkout nominal pago:**

| Ações do Ator | Ações do Sistema |
|---------------|------------------|
| | 1. Cria conta com dados do responsável pelos ingressos |

**Checkout nominal gratuito:**

| Ações do Ator | Ações do Sistema |
|---------------|------------------|
| | 1. Cria conta com dados do responsável pelos ingressos |

**Checkout não nominal pago:**

| Ações do Ator | Ações do Sistema |
|---------------|------------------|
| | 1. Cria conta com dados do responsável pelos ingresso |

**Checkout não nominal gratuito:**

| Ações do Ator | Ações do Sistema |
|---------------|------------------|
| | 1. Usa conta do usuário já autenticado |

### Cenário de Exceção

**E-mail já cadastrado:**

| Ações do Ator | Ações do Sistema |
|---------------|------------------|
| | 1. Não cria nova conta |
| | 2. Usa conta existente |
| | 3. Vincula ingressosaoaconta |

### Requisitos Não-Funcionais

- Enviar 2 e-mails distintos: boas-vindas + ingressos
- Não duplicar contas existentes
