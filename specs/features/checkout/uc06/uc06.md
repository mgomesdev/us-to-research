## UC06 - Finalizar Checkout com Cartão de Crédito

- **Caso de Uso Geral:** Checkout
- **Ator Principal:** Participante
- **Atores Secundários:** Sistema de Pagamento (Pay)
- **Resumo:** Este caso de uso representa o processo de pagamento com cartão de crédito. O sistema processa o pagamento e aguarda resposta da operadora.
- **Pré-condições:** Dados do comprador preenchidos e método de pagamento selecionado
- **Pós-condições:** Pagamento aprovado ou recusado

### Cenário Principal

| Ações do Ator | Ações do Sistema |
|---------------|------------------|
| 1. Seleciona "Cartão de crédito" | 2. Exibe campos do cartão |
| 3. Preenche dados do cartão | |
| 4. Clica em "Confirmar compra" | 5. Envia dados para processamento |
| | 6. Aguarda resposta da operadora |
| | 7. Pagamento aprovado: exibe tela de confirmação |
| | 8. Pagamento recusado: retorna ao Bloco 3 mantendo dados |

### Cenário Alternativo

**Cartão recusado:**

| Ações do Ator | Ações do Sistema |
|---------------|------------------|
| | 1. Mantém todos os dados preenchidos |
| | 2. Exibe mensagem de erro da operadora |
| 3. Tenta novamente ou altera método | |

**Usuário logado com cartão salvo:**

| Ações do Ator | Ações do Sistema |
|---------------|------------------|
| | 1. Exibe cartão salvo como opção |
| 2. Preenche apenas CVV | 3. Processa pagamento |

### Cenário de Exceção

- Erro de comunicação: exibir mensagem genérica e opção de retry
- Timeout: tratar como exceção e permitir nova tentativa

### Requisitos Não-Funcionais

- Manter dados preenchidos em caso de falha
- Exibir instruções claras em caso de erro
