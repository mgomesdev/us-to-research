## UC06 - Finalizar compra

- **Ator Principal:** Participante
- **Atores Secundários:** Sistema de Pagamento
- **Resumo:** Este caso de uso representa o processo final de conclusão do checkout, onde o participante confirma a compra após preencher todos os dados obrigatórios nos três blocos. O sistema processa o pagamento conforme o método selecionado e cria conta automaticamente quando aplicável.
- **Pré-condições:** Todos os campos obrigatórios dos três blocos preenchidos e validados.
- **Pós-condições:** Pagamento processado, conta criada (se aplicável) e transição para tela de confirmação ou status de pagamento pendente.

### Cenário Principal

| Ações do Ator | Ações do Sistema |
|---------------|------------------|
| 1. Clica em "Avançar para pagamento" | 2. Valida todos os blocos |
| | 3. Se válido: processa conforme método |
| | 4. Inclui: UC07 - Criar conta automaticamente |
| | 5. Inclui: UC08/UC09/UC10 - Processar pagamento |
| | 6. Inclui: UC11 - Exibir tela de confirmação |

### Restrições/Validações

- Botão "Avançar para pagamento" só é habilitado quando todos os campos obrigatórios estão válidos
- Tempo máximo para conclusão: 15 minutos (contador regressivo)

### Cenário Alternativo

- **Checkout gratuito:** Exibe apenas botão de confirmação de inscrição

### Cenário de Exceção

- Tempo expirado: sistema redireciona para página do evento com ingressos liberados
- Pagamento recusado: sistema retorna ao bloco de pagamento mantendo dados
