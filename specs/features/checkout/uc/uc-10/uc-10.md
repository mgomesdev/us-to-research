## UC10 - Processar pagamento via boleto

- **Ator Principal:** Participante
- **Atores Secundários:** Sistema de Pagamento
- **Resumo:** Este caso de uso representa o processo de processamento de pagamento via boleto bancário. O sistema gera e exibe o boleto para o participante realizar o pagamento. A confirmação ocorre após compensação do boleto.
- **Pré-condições:** Método de pagamento "Boleto" selecionado e dados do comprador preenchidos.
- **Pós-condições:** Boleto gerado e exibido, ou confirmação de pagamento recebida (transição para tela de confirmação).

### Cenário Principal

| Ações do Ator | Ações do Sistema |
|---------------|------------------|
| 1. Seleciona método Boleto | 2. Exibe opção Boleto |
| 3. Confirma pagamento | 4. Gera boleto |
| | 5. Exibe tela "Aguardando pagamento" |
| | 6. Permite download do boleto |
| | 7. Permite copiar código do boleto |
| | 8. Aguarda confirmação após compensação |
| | 9. Recebe confirmação: exibe tela de confirmação |

### Restrições/Validas

- Tempo para pagamento: padrão de vencimento do boleto
- Considera tempo de compensação do meio de pagamento

### Cenário Alternativo

- **Boleto pago após vencimento:** Sistema trata conforme regras de negócio do evento

### Cenário de Exceção

- Falha na geração do boleto: sistema exibe erro e permite retry
- Boleto não compensado: sistema notifica participant sobre status
