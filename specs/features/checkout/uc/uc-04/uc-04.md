## UC04 - Preencher dados do comprador

- **Ator Principal:** Participante
- **Atores Secundários:**
- **Resumo:** Este caso de uso representa o processo por meio do qual o participante informa ou confirma os dados do comprador. O sistema exibe um dropdown para selecionar o responsável pelo pagamento, que pode ser um dos participantes ou "Outro". Quando há dados salvos de compras anteriores (usuário logado), o sistema pré-preenche automaticamente.
- **Pré-condições:** Método de pagamento selecionado.
- **Pós-condições:** Dados do comprador validados e prontos para processamento do pagamento.

### Cenário Principal

| Ações do Ator | Ações do Sistema |
|---------------|------------------|
| 1. Visualiza dropdown "Responsável pelo pagamento" | 2. Exibe opções: participantes do Bloco 1 + "Outro" |
| 3. Seleciona opção | 4. Se participante: pré-preenche com dados do Bloco 1 |
| | 5. Se "Outro": exibe campos para preenchimento manual |
| | 6. Se usuário logado com dados salvos: pré-preenche automaticamente |
| 7. Preenche/edita dados do comprador | 8. Valida campos em tempo real |
| 9. (Opcional) Marca "Salvar dados para futuras compras" | 10. Armazena dados se marcado |
| 11. Confirma dados | 12. Valida todos os campos obrigatórios |

### Restrições/Validações

- Campo é independente do "Responsável pelosingressos" do Bloco 2
- Dados a serem salvos: nome completo, e-mail, CPF/CNPJ/Passaporte, telefone
- Dados do cartão NÃO são armazenados (apenas card_id)
- Dados são vinculados ao usuário, não ao método de pagamento
- Usuário pode editar qualquer campo mesmo com pré-preenchimento

### Cenário Alternativo

- **Usuário logado com dados salvos:** Sistema pré-preenche automaticamente. Usuário pode editar.
- **Usuário desmarca "Salvar dados":** Sistema limpa dados salvos para próximas compras.

### Cenário de Exceção

- Dados inválidos: sistema exibe mensagem de erro específica
- Campos obrigatórios não preenchidos: sistema exibe indicação visual
