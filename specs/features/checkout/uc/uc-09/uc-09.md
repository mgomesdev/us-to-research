## UC09 - Processar pagamento via Pix

- **Ator Principal:** Participante
- **Atores Secundários:** Sistema de Pagamento
- **Resumo:** Este caso de uso representa o processo de processamento de pagamento utilizando Pix. O sistema gera e exibe o QR Code e o código Pix copia e cola para o participante realizar o pagamento. A confirmação ocorre via webhook do sistema financeiro.
- **Pré-condições:** Método de pagamento "Pix" selecionado e dados do comprador preenchidos.
- **Pós-condições:** QR Code gerado e exibido, ou confirmação de pagamento recebida (transição para tela de confirmação).

### Cenário Principal

| Ações do Ator | Ações do Sistema |
|---------------|------------------|
| 1. Seleciona método Pix | 2. Exibe opções Pix |
| 3. Confirma pagamento | 4. Gera QR Code e código Pix |
| | 5. Exibe QR Code e código copia e cola |
| | 6. Exibe tempo restante para pagamento |
| | 7. Aguarda confirmação via webhook |
| | 8. Recebe confirmação: exibe tela de confirmação |

### Restrições/Validações

- Tempo para pagamento: conforme configurado no sistema (padrão 15 minutos)
- Tela de confirmação completa exibida apenas após recebimento do webhook de pagamento confirmado
- Participante pode copiar o código ou escanear QR Code

### Cenário Alternativo

- **Pagamento não realizado no tempo limite:** Sistema cancela transação e redireciona para checkout

### Cenário de Exceção

- Timeout ou falha no webhook: sistema permanece em等待状态 com opção de verificar manualmente
- Erro na geração do Pix: sistema exibe erro e permite retry
