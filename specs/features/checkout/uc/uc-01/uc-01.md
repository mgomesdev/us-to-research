## UC01 - Preencher dados dos participantes

- **Ator Principal:** Participante
- **Atores Secundários:** Organizador (fornece campos customizados)
- **Resumo:** Este caso de uso representa o processo por meio do qual o participante informa os dados necessários para cada ingresso selecionado no checkout. No checkout nominal, o sistema exibe um card para cada ingresso contendo campos obrigatórios (nome, e-mail, celular com DDI). Campos adicionais customizados podem ser configurados pelo organizador. No checkout não nominal, este bloco não é exibido.
- **Pré-condições:** Participante deve ter seleccionado pelo menos um ingresso para o evento.
- **Pós-condições:** Dados dos participantes validados e armazenados temporariamente para uso nos blocos seguintes.

### Cenário Principal

| Ações do Ator | Ações do Sistema |
|---------------|------------------|
| 1. Visualiza card de preenchimento para cada ingresso selecionado | 2. Exibe card identificado por tipo e valor do ingresso |
| 3. Preenche nome do participante | 4. Valida campo em tempo real |
| 5. Preenche e-mail do participante | 6. Valida formato de e-mail |
| 7. Preenche celular com DDI | 8. Exibe seletor de DDI com Brasil pré-selecionado |
| 9. Preenche campos customizados (se disponíveis) | 10. Aplica regras de obrigatoriedade definidas pelo organizador |
| 11. Clica em botão para avançar | 12. Valida todos os campos obrigatórios |
| | 13. Se válidos, permite avanço. Se inválidos, exibe mensagem de erro |

### Restrições/Validações

- Campos obrigatórios: nome, e-mail, celular (com DDI)
- E-mail deve ter formato válido
- Celular deve conter DDI com bandeira do país
- Campos customizados seguem obrigatoriedade configurada pelo organizador
- Todos os campos devem ser validados antes do avanço para próximo bloco

### Cenário Alternativo

- **Checkout não nominal:** O Bloco 1 não é exibido. O caso de uso não é executado.

### Cenário de Exceção

- Campo obrigatório não preenchido: sistema exibe indicação visual de obrigatoriedade e mensagem explicativa
- Dados inválidos: sistema exibe mensagem de erro específica para cada tipo de invalidade
- Timeout de sessão: dados são perdidos e participante precisa reiniciar o preenchimento
