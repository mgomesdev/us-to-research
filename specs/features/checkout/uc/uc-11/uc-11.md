## UC11 - Exibir tela de confirmação

- **Ator Principal:** Participante
- **Atores Secundários:**
- **Resumo:** Este caso de uso representa a tela exibida ao participante após a conclusão bem-sucedida do checkout. A tela de confirmação exibe os detalhes da compra e confirma a inscrição ou aquisição dos-ingressos.
- **Pré-condições:** Pagamento aprovado ou checkout gratuito confirmado.
- **Pós-condições:** Tela de confirmação exibida, e-mails transacionais enviados.

### Cenário Principal

| Ações do Ator | Ações do Sistema |
|---------------|------------------|
| | 1. Recebe confirmação de pagamento aprovado |
| | 2. Exibe tela de confirmação (mesma já existente) |
| | 3. Envia e-mail com-ingressos em PDF |
| | 4. Envia e-mail de confirmação de compra |
| | 5. Se conta criada automaticamente: envia e-mail de boas-vindas |

### Restrições/Validações

- Tela de confirmação: mesma já existente no sistema atualmente
- E-mails enviados após confirmação do pagamento

### Cenário Alternativo

- **Checkout gratuito:** Exibe tela de confirmação de inscrição

### Cenário de Exceção

- Falha no envio de e-mail: sistema registra erro mas não impede conclusão do checkout
