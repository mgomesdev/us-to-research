## UC07 - Criar conta automaticamente

- **Ator Principal:** Participante
- **Atores Secundários:**
- **Resumo:** Este caso de uso representa o processo de criação automática de conta na plataforma Linka Eventos após a conclusão bem-sucedida do checkout. A conta é criada usando os dados do responsável pelosingressos definido no Bloco 2.
- **Pré-condições:** Checkout nominal ou não nominal pago concluído com sucesso.
- **Pós-condições:** Conta criada no sistema, responsável pelosingressos vinculado à conta, e-mails de boas-vindas e-ingressos enviados.

### Cenário Principal

| Ações do Ator | Ações do Sistema |
|---------------|------------------|
| | 1. Recebe dados do responsável pelosingressos do Bloco 2 |
| | 2. Verifica se e-mail já existe na base |
| | 3. Se existe: usa conta existente |
| | 4. Se não existe: cria nova conta com dados fornecidos |
| | 5. Envia e-mail de boas-vindas com link para definir senha |
| | 6. Após pagamento confirmado: envia e-mail com-ingressos |

### Restrições/Validações

- **Checkout nominal pago:** Cria conta com dados do responsável (participante selecionado ou "Outro")
- **Checkout nominal gratuito:** Cria conta automaticamente
- **Checkout não nominal pago:** Cria conta com dados do responsável
- **Checkout não nominal gratuito:** Exige autenticação prévia (não cria conta)
- Se e-mail já existe: usa conta existente sem criar duplicata

### Cenário Alternativo

- **E-mail já cadastrado:** Sistema vincula-ingressos à conta existente, não cria nova conta

### Cenário de Exceção

- Falha na criação de conta: sistema notifica erro e permite retry
- E-mail inválido: sistema rejeita e solicita correção
