## UC08 - Processar pagamento via cartão

- **Ator Principal:** Participante
- **Atores Secundários:** Sistema de Pagamento
- **Resumo:** Este caso de uso representa o processo de processamento de pagamento utilizando cartão de crédito. O sistema envia os dados do cartão para a operadora e aguarda a resposta para approval ou recusa.
- **Pré-condições:** Método de pagamento "Cartão de crédito" selecionado e dados do comprador preenchidos.
- **Pós-condições:** Pagamento aprovado (transição para confirmação) ou recusado (retorno ao bloco de pagamento com erro).

### Cenário Principal

| Ações do Ator | Ações do Sistema |
|---------------|------------------|
| 1. Seleciona método cartão de crédito | 2. Exibe campos do cartão |
| 3. Preenche dados do cartão | 4. Valida formato dos dados |
| 5. Confirma pagamento | 6. Envia dados para operadorado |
| | 7. Aguarda resposta |
| | 8. Se aprovado: exibe tela de confirmação |
| | 9. Se recusado: retorna ao bloco de pagamento |

### Restrições/Validações

- Dados do cartão: número, validade, CVV, nome do titular
- Sistema armazena apenas card_id retornado pelo pagamento (não armazena dados sensíveis)
- Se usuário logado com cartão salvo: exibe cartão salvo, exige apenas CVV

### Cenário Alternativo

- **Cartão recusado:** Sistema mantém todos os dados preenchidos, exibe mensagem de erro da operadora com instruções. Usuário pode tentar novamente ou alterar método.

### Cenário de Exceção

- Timeout na comunicação com operadora: sistema exibe erro genérico e permite retry
- Erro na integração: sistema notifica erro técnico
